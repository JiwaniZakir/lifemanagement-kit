# Schwab Investments

Monitor investment portfolios via [Schwab](https://developer.schwab.com/). Two-step trade execution with preview and confirmation for safety.

## Feasibility

Partial support. The `schwab-py` library provides read access to accounts and positions. Trading requires OAuth approval from Schwab, which may involve additional verification.

## Setup

### 1. Register a Schwab Developer App

1. Sign up at [developer.schwab.com](https://developer.schwab.com/)
2. Create an application
3. Note your **App Key** and **App Secret**
4. Set your callback URL (typically `https://127.0.0.1`)

### 2. Configure Environment Variables

Add to your `.env` file:

```bash
SCHWAB_APP_KEY=your_app_key
SCHWAB_APP_SECRET=your_app_secret
SCHWAB_CALLBACK_URL=https://127.0.0.1
```

### 3. Authorize via OAuth

The Schwab integration uses OAuth 2.0. The initial authorization flow:

1. Call `GET /finance/schwab/auth` to get the authorization URL
2. Complete the OAuth flow in a browser
3. The callback exchanges the code for tokens, which are encrypted and stored

## Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/finance/schwab/auth` | GET | Start OAuth authorization flow |
| `/finance/schwab/portfolio` | GET | Retrieve portfolio positions and balances |
| `/finance/portfolio` | GET | Combined portfolio view |
| `/finance/schwab/trade/preview` | POST | Preview a trade (returns cost, fees, impact) |
| `/finance/schwab/trade/confirm` | POST | Confirm a previewed trade with approval token |

## Integration Client

The Schwab client lives at `data-api/app/integrations/schwab_client.py` and handles:

- OAuth 2.0 token exchange and refresh
- Account and position retrieval
- Two-step trade execution pattern:
  1. **Preview**: shows the trade details, cost, and potential impact
  2. **Confirm**: requires an approval token from the preview step

## Trade Safety

Trades use a two-step confirmation pattern to prevent accidental execution:

1. `POST /finance/schwab/trade/preview` returns trade details and an approval token
2. `POST /finance/schwab/trade/confirm` requires that approval token to execute
3. The agent always shows the preview to the user and waits for explicit confirmation

The `aegis-finance` skill handles Schwab-related queries:

- Portfolio position monitoring
- Account balance summaries
- Trade preview and confirmation workflow
- All account numbers masked to last 4 digits

## Notes

- The `schwab-py` library is guarded with `try/except ImportError`. If not installed, the integration gracefully degrades.
- Trading capabilities require explicit OAuth approval from Schwab. Read-only access is available by default.
- OAuth refresh tokens are encrypted and stored via the credential store. Token refresh is handled automatically by the client.
