# Production Deployment Guide

This guide covers deploying Aegis to a production VPS. If you are running locally for development, see [DEVELOPMENT.md](DEVELOPMENT.md) instead.

## 1. VPS Setup

Aegis runs on any Linux host with Docker. Recommended:

- **Hetzner Cloud CPX31+** (4 vCPU, 8 GB RAM) or equivalent
- Ubuntu 22.04+ or Debian 12+
- At least 20 GB disk (PostgreSQL data + Docker images)
- A domain name pointed at Cloudflare (for tunnel access)

Minimum requirements: 2 CPU cores, 4 GB RAM.

## 2. Install Docker and Docker Compose

```bash
# Install Docker (official script)
curl -fsSL https://get.docker.com | sh

# Add your user to the docker group
sudo usermod -aG docker $USER

# Verify Docker Compose v2 is available (bundled with Docker)
docker compose version
# Expected: Docker Compose version v2.29.x or later
```

Log out and back in for the group change to take effect.

## 3. Clone and Bootstrap

```bash
git clone https://github.com/JiwaniZakir/aegis.git
cd aegis

# Bootstrap generates secrets, starts services, runs migrations
./infrastructure/scripts/bootstrap.sh
```

Bootstrap does the following:
1. Copies `.env.example` to `.env`
2. Generates `DATA_API_TOKEN`, `ENCRYPTION_MASTER_KEY`, and `POSTGRES_PASSWORD` using `openssl rand`
3. Starts all Docker Compose services
4. Waits for the data-api health check to pass
5. Runs Alembic database migrations

## 4. Configure Environment Variables

Edit `.env` with your actual values. The required variables are:

| Variable | Required | Description |
|----------|----------|-------------|
| `ANTHROPIC_API_KEY` | Yes | Your Claude API key from Anthropic |
| `DATA_API_TOKEN` | Auto | Machine-to-machine auth token (generated by bootstrap) |
| `ENCRYPTION_MASTER_KEY` | Auto | AES-256 encryption key (generated by bootstrap) |
| `POSTGRES_PASSWORD` | Auto | Database password (generated by bootstrap) |
| `OPENCLAW_GATEWAY_TOKEN` | Auto | Gateway admin token (generated by bootstrap) |
| `WHATSAPP_ADMIN_PHONE` | No | Your phone number with country code (e.g., `15551234567`) |
| `CLOUDFLARE_TUNNEL_TOKEN` | No | Cloudflare Tunnel token (production only) |

Integration-specific variables are documented in `.env.example`. Only add the ones you plan to use.

After editing `.env`, restart services:

```bash
docker compose restart openclaw-gateway
```

## 5. Set Up Cloudflare Tunnel

Cloudflare Tunnel provides zero-trust access to your Aegis instance without opening any ports on your VPS.

### 5.1 Create a tunnel

1. Log in to the [Cloudflare Zero Trust dashboard](https://one.dash.cloudflare.com/)
2. Navigate to **Networks > Tunnels**
3. Click **Create a tunnel**
4. Choose **Cloudflared** as the connector
5. Name your tunnel (e.g., `aegis`)
6. Copy the tunnel token

### 5.2 Configure the tunnel

Add the token to `.env`:

```bash
CLOUDFLARE_TUNNEL_TOKEN=<your-tunnel-token>
```

### 5.3 Set up DNS routing

In the Cloudflare dashboard, add a public hostname for your tunnel:

- **Subdomain**: `aegis` (or your preferred name)
- **Domain**: Your Cloudflare-managed domain
- **Service**: `http://openclaw-gateway:18789`

This routes `aegis.yourdomain.com` through the tunnel to your OpenClaw gateway.

### 5.4 (Optional) Access policies

In the Cloudflare Zero Trust dashboard, create an access policy to restrict who can reach your tunnel:

1. Go to **Access > Applications**
2. Add an application for your tunnel hostname
3. Set a policy (e.g., email allowlist with your email address)

## 6. Deploy to Production

```bash
# Full production deployment (includes SOPS decryption, validation, build, health checks)
make prod

# Or run the script directly:
./infrastructure/scripts/deploy.sh --prod
```

The deploy script:
1. Decrypts any SOPS-encrypted secret files
2. Validates that all required `.env` variables are set
3. Ensures the OpenClaw gateway repository is cloned
4. Builds and starts all services with production overrides
5. Waits for health checks to pass on all services
6. Cleans up decrypted secret files on exit

Production overrides (`docker-compose.prod.yml`) add:
- Resource limits (memory, CPU) for each service
- `restart: always` for all services
- Log rotation (`json-file` driver, 50 MB max, 5 files)
- Read-only filesystem for data-api
- `ENVIRONMENT=production` and 4 Uvicorn workers for data-api

## 7. Verify Health Checks

```bash
# Check all service health
make health

# Or manually:
docker compose ps

# Individual service checks:
docker compose exec data-api curl -sf http://127.0.0.1:8000/health
docker compose exec openclaw-gateway node -e 'require("http").get("http://localhost:18789/health",(r)=>{process.exit(r.statusCode===200?0:1)}).on("error",()=>process.exit(1))'
docker compose exec postgres pg_isready -U aegis
```

All services should show `Up (healthy)`.

## 8. Set Up Backups

### Manual backup

```bash
make backup
# Or:
./infrastructure/scripts/backup.sh
```

This dumps PostgreSQL, compresses with gzip, encrypts with `age` (if installed), and stores in `backups/`. Old backups beyond the last 7 are automatically deleted.

### Automated backups via cron

```bash
# Edit crontab
crontab -e

# Add daily backup at 3 AM
0 3 * * * cd /path/to/aegis && ./infrastructure/scripts/backup.sh >> /var/log/aegis-backup.log 2>&1
```

### Restore from backup

```bash
# If encrypted with age:
age -d -i /path/to/age-key backups/aegis_20260301_030000.sql.gz.age | gunzip | \
  docker compose exec -T postgres psql -U aegis aegis

# If unencrypted:
gunzip -c backups/aegis_20260301_030000.sql.gz | \
  docker compose exec -T postgres psql -U aegis aegis
```

### Off-site backup (recommended)

Copy encrypted backups to a remote location:

```bash
# Example: rsync to another server
rsync -avz backups/*.age user@backup-server:/backups/aegis/

# Example: upload to S3-compatible storage
aws s3 sync backups/ s3://your-bucket/aegis-backups/ --include "*.age"
```

## 9. Secret Rotation

Rotate internal secrets every 90 days:

```bash
./infrastructure/scripts/rotate-secrets.sh
```

This rotates `POSTGRES_PASSWORD` and `DATA_API_TOKEN`, updates `.env`, updates the running PostgreSQL password, and restarts all services.

To rotate the encryption key (requires re-encrypting all stored credentials):

```bash
ROTATE_ENCRYPTION_KEY=true ./infrastructure/scripts/rotate-secrets.sh
```

**Warning:** Rotating the encryption key without re-encrypting credentials will make them permanently unreadable. Follow the script's output carefully.

External API keys (Anthropic, Plaid, Google, etc.) must be rotated manually through their respective dashboards.

## 10. Monitoring

### Logs

```bash
# Follow all service logs
make logs

# Specific service logs
docker compose logs -f openclaw-gateway
docker compose logs -f data-api
docker compose logs -f postgres

# Last 100 lines from a specific service
docker compose logs --tail=100 data-api
```

### Health endpoints

| Service | Endpoint | Method |
|---------|----------|--------|
| data-api | `http://127.0.0.1:8000/health` | GET (inside container) |
| OpenClaw gateway | `http://localhost:18789/health` | GET (inside container) |
| PostgreSQL | `pg_isready -U aegis` | CLI (inside container) |

### Audit log

Check audit log integrity:

```bash
# From the host (dev mode with port bindings):
curl -s -H "Authorization: Bearer $DATA_API_TOKEN" http://localhost:8000/audit/verify

# From inside the Docker network:
docker compose exec data-api curl -s -H "Authorization: Bearer $DATA_API_TOKEN" http://127.0.0.1:8000/audit/verify
```

### LLM budget

Check current LLM spend:

```bash
curl -s -H "Authorization: Bearer $DATA_API_TOKEN" http://localhost:8000/budget/usage
```

## 11. Updating

```bash
cd /path/to/aegis

# Pull latest changes
git pull origin main

# Rebuild and restart (the deploy script handles everything)
make prod

# Or for development:
docker compose up -d --build
```

If there are database schema changes, run migrations:

```bash
docker compose exec -T data-api uv run alembic upgrade head
```

## 12. Uninstalling

```bash
cd /path/to/aegis

# Stop and remove containers, networks
docker compose down

# Also remove volumes (DELETES ALL DATA)
docker compose down -v

# Remove images
docker rmi $(docker images 'aegis*' -q) 2>/dev/null
```

## Production Checklist

Before going live, verify:

- [ ] `ANTHROPIC_API_KEY` is set in `.env`
- [ ] `CLOUDFLARE_TUNNEL_TOKEN` is set (if using tunnel)
- [ ] `WHATSAPP_ADMIN_PHONE` is set (if using WhatsApp)
- [ ] All desired integrations are configured
- [ ] `make prod` completes with all services healthy
- [ ] WhatsApp is paired via the Control UI
- [ ] A backup runs successfully (`make backup`)
- [ ] Backup cron job is configured
- [ ] Secret rotation reminder is calendared (every 90 days)
- [ ] `docker-compose.override.yml` is NOT active (it is auto-loaded only in dev; production uses `deploy.sh --prod` which explicitly specifies compose files)
